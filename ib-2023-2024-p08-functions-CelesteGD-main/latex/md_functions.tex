\hypertarget{md_functions_autotoc_md5}{}\doxysubsubsection{Doxygen}\label{md_functions_autotoc_md5}
\href{https://en.wikipedia.org/wiki/Doxygen}{\texttt{ Doxygen}} es una herramienta de código abierto que permite generar documentación de referencia para proyectos de desarrollo software. Una ventaja de Doxygen es que la documentación está escrita en el propio código fuente de los programas, y por lo tanto es relativamente fácil de mantener actualizada. Doxygen puede hacer referencias cruzadas entre la documentación y el código, de modo que el lector de un documento puede referirse fácilmente al código fuente. La herramienta extrae la documentación de los comentarios presentes en los ficheros de código fuente y puede generar la salida en diferentes formatos entre los cuales están H\+T\+ML, P\+DF, La\+TeX o páginas {\ttfamily man} de Unix.

En {\itshape Informática Básica} no se propone un uso exhaustivo de Doxygen pero {\bfseries{sí se requiere que la documentación de los programas desarrollados se realice en el formato reconocido por Doxygen}}, que se ha convertido en un estándar de facto.

Comience por instalar Doxygen en su máquina virtual de la asignatura\+: 
\begin{DoxyCode}{0}
\DoxyCodeLine{\$ sudo apt install doxygen}
\end{DoxyCode}


Instale también los siguientes paquetes\+: 
\begin{DoxyCode}{0}
\DoxyCodeLine{\$ sudo apt install texlive-\/latex-\/base}
\DoxyCodeLine{\$ sudo apt install texlive-\/latex-\/recommended}
\DoxyCodeLine{\$ sudo apt install texlive-\/latex-\/extra}
\end{DoxyCode}


Estos paquetes son necesarios para compilar ficheros en formato \href{https://es.wikipedia.org/wiki/LaTeX}{\texttt{ La\+TeX}}. Más adelante en este documento se justifica la necesidad de los programas que suministran estos paquetes.

En el \href{https://www.doxygen.nl/manual/starting.html}{\texttt{ manual de Doxygen}} se indica cómo comenzar a trabajar con la herramienta. Si, ubicados en un directorio de trabajo, se invoca\+: 
\begin{DoxyCode}{0}
\DoxyCodeLine{doxygen -\/g <config-\/file>}
\end{DoxyCode}


la herramienta creará un fichero de configuración. Si no se le pasa el nombre del fichero ({\itshape config-\/file}) como parámetro, creará un fichero con nombre {\ttfamily Doxyfile} preconfigurado para su uso. En el directorio de trabajo de esta práctica ({\ttfamily src}) se encuentra un fichero {\ttfamily Doxyfile} ya listo para usarse con proyectos de C++. Se ha incluído asimismo en ese directorio el código fuente de un programa para ilustrar con el mismo el uso de documentación con Doxygen. Si revisa el fichero {\ttfamily Doxyfile} (es un fichero de texto) verá un conjunto de opciones que el programa permite. Cada opción va precedida de una explicación de su finalidad y funcionamiento, de modo que puede probar a modificar algunas de ellas si lo desea. En \href{https://www.doxygen.nl/manual/config.html}{\texttt{ esta página}} puede consultarse la finalidad y funcionamiento de cada una de las etiquetas (tags) que se usan en el fichero de configuración de Doxygen.

Para generar la documentación de su aplicación, colóquese en el directorio de su proyecto ({\ttfamily src} en esta práctica) y ejecute\+: 
\begin{DoxyCode}{0}
\DoxyCodeLine{doxygen Doxyfile}
\end{DoxyCode}


Con el fichero {\ttfamily Doxyfile} que se suministra, la herramienta creará un subdirectorio {\ttfamily doc} en el directorio raíz de su proyecto en el que alojará toda la documentación generada. El directorio donde Doxygen genera su salida se especifica con la etiqueta {\ttfamily O\+U\+T\+P\+U\+T\+\_\+\+D\+I\+R\+E\+C\+T\+O\+RY} (línea {\ttfamily 61} del fichero {\ttfamily Doxyfile} suministrado). Con la configuración suministrada se generan 2 subdirectorios dentro de {\ttfamily doc}\+: {\ttfamily html} y {\ttfamily latex}. Si se coloca en el directorio {\ttfamily doc/latex/} y ejecuta {\ttfamily make} el sistema \char`\"{}compila\char`\"{} el código La\+TeX y genera un fichero {\ttfamily refman.\+pdf} que contiene la documentación generada. Observe en este caso el uso de {\ttfamily make} con un fichero {\ttfamily Makefile} no para compilar un programa sino para generar un fichero {\ttfamily pdf} a partir del código (texto) La\+TeX. Si trabaja en su máquina virtual, traiga el fichero {\ttfamily refman.\+pdf} hacia su máquina local y visualice su contenido.

Si abre con un navegador el fichero {\ttfamily doc/html/index.\+html} accederá a la página principal de la documentación generada para el programa. En su máquina virtual no va a poder abrir un navegador para explorar los ficheros del directorio {\ttfamily doc/html}. Lo que ha de hacer es o bien generarlos en su instalación Linux local o bien traer el contenido del directorio {\ttfamily doc} desde su máquina virtual IaaS hacia su máquina local y en la máquina local abrir con un navegador el fichero {\ttfamily doc/html/index.\+html}.

Tal como se ha indicado, H\+T\+ML o La\+TeX son solo 2 de los formatos que permite generar Doxygen. Tanto H\+T\+ML como La\+TeX (también \href{https://es.wikipedia.org/wiki/Markdown}{\texttt{ Markdown}}) son lo que se conoce como \href{https://es.wikipedia.org/wiki/Lenguaje_de_marcado}{\texttt{ lenguajes de marcas}}. H\+T\+ML es el lenguaje que se utiliza para componer los textos que se muestran en las páginas web. \href{https://en.wikipedia.org/wiki/LaTeX}{\texttt{ Latex}} es un sistema de composición de textos que cuida el formato en especial en el ámbito de la tipografía y que es especialmente adecuado para textos de carácter científico. No se pretende aquí que profundice en conocer H\+T\+ML o La\+TeX.

La sección \href{https://www.doxygen.nl/manual/config.html}{\texttt{ Documenting the code}} del manual de Doxygen indica cómo comentar el código fuente de modo que los comentarios sean procesados por Doxygen para incorporarlos a la documentación generada.

La guía \href{https://developer.lsst.io/cpp/api-docs.html}{\texttt{ Documenting C++ Code}} de documentación de código del proyecto L\+L\+ST es la referencia que se adoptará en la asignatura para documentar el código de los programas que se desarrollen.
